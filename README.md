# GeoFS-Airport-taxiway
Supports taxiways at all airports but temporarily does not have the terminal part
This is a rendering effect that implements bright, matte, non-reflective taxiways for the GeoFS core, while integrating multiple Overpass API nodes, with dynamic loading and cache optimization: The rendering range is dynamically calculated based on the aircraft's current position, only loading taxiway data for the current area, and automatically clearing old data to avoid excessive memory usage; Scheduled refresh: Automatically detects the aircraft's position at regular intervals and updates taxiway data to ensure real-time synchronization of taxiway content as the aircraft moves. 1. Initialization and Startup: After the script loads, it initializes the taxiway data storage array and executes the first taxiway update after a 3-second delay to avoid conflicts with the GeoFS main program loading. 2. Dynamic Data Update: The aircraft's position is checked every 8 seconds. If the position is outside the current rendering boundary, old taxiway rendering data is automatically cleared and new area data is requested; if the position has not changed, the current data is directly reused to reduce unnecessary network requests.
